Guía práctica – Semana 2 Día 5

Actividad:
Formulario “Participa”, checklist de accesibilidad y reflexión de verificación responsable.

Repositorio:
Repositorio dispuesto por el docente.

Importante:
No debes trabajar directamente en la rama main.

Cada estudiante debe crear una rama personal con el formato:

nombres_apellidos

Ejemplo:
maria_gomez
juan_perez
laura_torres_garcia

Evidencia:
La evidencia de esta actividad es únicamente el push de tu rama personal a GitHub.

No debes entregar nada en Moodle.
No debes enviar capturas.
No debes subir archivos adicionales.
No debes publicar enlaces por otro medio.

Pasos obligatorios:

1. Clonar el repositorio

git clone URL_DEL_REPOSITORIO

2. Entrar a la carpeta del proyecto

cd NOMBRE_DEL_REPOSITORIO

3. Crear la rama personal

git checkout -b nombres_apellidos

4. Verificar la rama actual

git branch

La rama marcada con * debe ser tu rama personal, no main.

5. Modificar los archivos

semana2/index.html
semana2/styles.css
semana2/checklist-accesibilidad.txt
semana2/reflexion-verificacion.txt

6. Revisar cambios

git status

7. Agregar cambios

git add .

8. Crear commit

git commit -m "Entrega formulario participa semana 2"

9. Publicar la rama en GitHub

git push -u origin nombres_apellidos

10. Verificar en GitHub

Entra al repositorio y confirma que tu rama aparece publicada.
